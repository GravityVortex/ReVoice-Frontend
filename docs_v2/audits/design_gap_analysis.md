# Vozo 设计 vs 当前实现分析

基于 `docs_v2/audits/vozo_img` 中的图片与 `src/app/[locale]/(landing)/video_convert/video-editor/[id]` 下当前代码的对比，以下是主要的差异和可借鉴的优化点。

## 1. 整体布局与方向

| 功能特性 | Vozo 设计 (目标) | 当前实现 | 差距 / 借鉴点 |
| :--- | :--- | :--- | :--- |
| **主布局** | **左侧/中间**: 字幕编辑器 (源文 & 译文)<br>**右侧**: 视频预览<br>**最左侧**: 工具侧边栏 | **左侧**: 视频预览 + 时间轴<br>**右侧**: 字幕列表面板 | **布局反转**: 当前布局优先展示左侧视频，而 Vozo 把字幕内容作为核心放在中间/左侧。Vozo 的视频作为次要（但较大）的参考放在右侧。 |
| **时间轴** | **底部全宽**: 横跨编辑器整个宽度，方便对长时间轴进行精细控制。 | **嵌套在视频面板内**: 被限制在左侧面板底部 (约 60% 宽度)，显得局促。 | **行动**: 将时间轴移出分割面板，放置在屏幕底部固定区域，横跨全屏。 |
| **工具侧边栏** | **独立侧边栏**: 最左侧的垂直图标栏 (“编辑语言”, “字幕”, “品牌” 等)。 | **内嵌按钮**: 工具栏按钮混合在视频面板内或分散各处。 | **行动**: 在左侧引入一个纤细的垂直导航栏，用于高层级的功能模式切换。 |

## 2. 字幕编辑器 ("工作台")

| 功能特性 | Vozo 设计 | 当前实现 | 差距 / 借鉴点 |
| :--- | :--- | :--- | :--- |
| **视觉层级** | **主工作区**: 占据屏幕大部分空间的双列大表格 (原文 vs 译文)。 | **次要面板**: 被降级为右侧侧边栏中的一个 "列表" (`AudioListPanel`)。 | **行动**: 提升字幕编辑器的地位，使其成为视觉焦点。使用整洁的双列布局，打造类似文档编辑器的体验，而不是设置列表。 |
| **发言人标识** | 每行显著的 "Speaker 1" 徽章。 | 存在但视觉融合度不够。 | **行动**: 优化发言人标签样式，采用 Vozo 图片中那种高级感的 "胶囊" 样式。 |

## 3. 加载与处理状态

| 功能特性 | Vozo 设计 | 当前实现 | 差距 / 借鉴点 |
| :--- | :--- | :--- | :--- |
| **视觉效果** | **沉浸式遮罩**: 深色简洁背景，中央有脉冲式 Logo 和清晰的 "步骤 X/Y" 进度展示。 | **骨架屏**: 模仿 UI 结构的通用灰色骨架条。 | **行动**: 实现一个专门的 `ProcessingState` 组件。相比骨架屏，这能建立期待感，并显得更精致。 |
| **用户反馈** | "完成后通知我" 开关。 | 无。 | **行动**: 增加长耗时任务的通知开关。 |

## 4. 时间轴细节

| 功能特性 | Vozo 设计 | 当前实现 | 差距 / 借鉴点 |
| :--- | :--- | :--- | :--- |
| **轨道** | 标签清晰的 "原文"、"译文" 视觉轨道。 | 现有轨道功能正常，但可以借鉴 Vozo 设计中更清晰、高对比度的颜色。 | **行动**: 优化轨道配色 (蓝/紫主题)，确保波形图清晰可见。 |

## 建议总结

1.  **重构布局结构**:
    -   **顶部**: 顶栏 (保持不变)。
    -   **中间**: 分割视图 -> **左侧**: 字幕编辑器 (主区) | **右侧**: 视频预览。
    -   **最左侧**: 增加编辑器工具导航栏。
    -   **底部**: 全宽时间轴。
2.  **升级加载界面**: 用 "三角 Logo" 步骤式处理界面替换骨架屏。
3.  **视觉打磨**: 采用深色主题，并使用紫色 (`#Primary`) 作为激活状态和轨道项的强调色。
