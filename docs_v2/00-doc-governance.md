# 文档治理（v2 口径）

> 最后校验：2026-03-04  
> 目标：保证文档“可维护、可检索、不漂移”。

---

## 1) 单一入口原则

- v2 文档的唯一入口：`docs_v2/README.md`
- 任何需要长期维护的规范/手册，必须落在 `docs_v2/` 下，并在索引中可被找到。

---

## 2) 文档类型与存放位置

- 现行规范/手册：直接放在 `docs_v2/`（或其子目录）
- 深度资料（仍需维护）：`docs_v2/architecture` / `docs_v2/howto` / `docs_v2/integration`
- 历史记录（默认不维护）：统一收敛在 `docs_v2/plans/archive/` 与 `docs_v2/audits/`；仓库不再保留旧 `docs/` 目录

---

## 3) 每篇文档必须包含的信息

建议在文档开头包含：

- `最后校验：YYYY-MM-DD`
- 单一来源/权威来源（如“以 `schema.ts` 为准”“以某 API 返回为准”）

---

## 4) 变更规则（避免漂移）

- 代码改动如果改变了“契约”（数据表/API/鉴权/计费/部署），必须同步更新对应 v2 文档
- 评审时如果发现文档与代码不一致，以代码为准，文档立即修正
