# 视频翻译三层页面渐进式 UI 设计

## 目标

围绕视频翻译工作流，建立一条清晰的三层页面递进关系：

1. 外部视频页：内容入口
2. 内部任务列表页：任务入口
3. 任务详情页：操作入口

整体视觉语言统一，但页面布局形态和信息密度必须逐层变化，让用户自然地从“看内容”过渡到“管理任务”，最后进入“处理结果”。

## 页面关系

```text
外部视频卡片页 → 内部任务列表页 → 任务详情工作台
```

### 外部视频卡片页

- 角色：内容入口
- 用户意图：先被视频内容吸引，再感知它是否有关联任务
- 布局形态：宽卡
- 信息优先级：
  - 封面
  - 标题
  - 一句亮点或简述
  - 轻量状态提示
- 设计原则：
  - 状态存在，但不能抢封面
  - 不展示详细进度、不展示复杂操作
  - 用户点击动机是“进来继续看看”

### 内部任务列表页

- 角色：任务入口
- 用户意图：快速判断任务现在是什么状态，是否要进入详情
- 布局形态：列表行，而不是继续用内容卡片
- 信息优先级：
  - 小缩略图
  - 任务标题
  - 语言对 / 时间 / 时长 / 积分
  - 进度或结果状态
  - 进入详情的明确入口
- 设计原则：
  - 视觉更收敛、更工具化
  - 状态标签前置
  - 一屏支持快速比较多条任务

### 任务详情页

- 角色：工作台
- 用户意图：确认视频内容，直接处理交付物
- 布局形态：左视频、右信息工作台
- 信息优先级：
  - 左侧大预览
  - 右侧时间信息
  - 成品视频 / 音频 / 字幕三张核心卡
- 设计原则：
  - 视频区整体贴近容器
  - 右侧只保留关键交付信息和动作
  - 视觉重心稳定，不再让用户重新理解状态

## 统一语言

三层页面统一以下内容：

- 状态颜色体系：处理中 / 已完成 / 失败
- 圆角和边框体系
- 暗色背景层次
- 缩略图与视频封面的视觉语言
- 交互方向：点击进入下一层，而不是在上一层解决下一层的问题

## 必须拉开的差异

- 外部视频卡片页：更轻、更展陈、更内容导向
- 内部任务列表页：更紧凑、更结构化、更状态导向
- 任务详情页：更稳定、更沉浸、更操作导向

## 成功标准

- 用户能明显感知三个页面属于同一系统，但不是重复套壳
- 从外部视频页进入任务页时，能感受到“从内容世界进入任务世界”
- 从任务列表页进入详情页时，能感受到“从判断状态进入处理结果”
- 三层页面的状态表达一致，但信息密度逐层递进
